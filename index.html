Posted on 2017-08-27 | In 讀書筆記 |

背景

  • 函式是 JavaScript 第一級物件（first-class object）。
  • 函式提供作用域。
  • 函式也是物件。

釐清術語的差異

  • 具名函式表示式（named function expression）。
  • 匿名函式（anonymous function）。
  • 函式宣告式（function declaration）

宣告式 vs 表示式：命名與 Hoisting

函式的 name 屬性

函式的 Hoisting

回呼模式

  • 將函式當成參數傳遞給其他的函式呼叫，簡稱回呼（callback）。

回呼與作用域

  • 如果回呼的方法使用 this 參考所屬物件，會發生非預期的行為。
  • 必須把 this 所屬的物件也傳入回呼函式的參數中。
  • 改用 call() 或 apply() 的方式來榜定呼叫。

非同步事件監聽器

  • 事件監聽器（event listener）是一個回呼函式指標。

逾時

  • setTimeout() 及 setInterval()

回傳函式

  • 函式回傳一個函式。
  • 產生閉包（closure），可以儲存一些 private 資料。

自我定義的函式

立即函式

  • 定義函式時立即執行
  • 提供一種作用域的 沙盒 給初始化的程式碼。

立即函式的參數

  • 一般使用全域物件作為參數傳遞，函式內部就可存取。
  • 不該傳遞太多參數

立即函式的回傳值

  • 可利用立即函式的作用域儲存 private 資料。
  • 可用來定義屬性

優點及用法

  • 將運算包裝起來，不必留下全域變數。
  • 功能模組化

立即物件初始化

  • 使用物件實字定義物件之後在立即執行所定義的初始化函式

初始階段的分支

  • 初始階段（Init-time），也稱為讀取階段（load-time）
  • 某個條件，在整個程式生命週期都不會改變，只檢查一次。
  • 瀏覽器偵測或功能檢測的例子。

函式屬性 - 記憶模式

  • 函式可以新增屬性，可以用來儲存某個運算候的結果，下次在呼叫時就不需要再運算
  • function 使用 cache 的例子。

設定值物件

  • 一個函式的參數使用一個物件參數當作設定值物件。

Curry

函式的應用

  • 函式應用（applied），可以使用 Function.prototype.apply() 方法來應用一個函式

部份應用

Currying

  • 一種函式的轉換過程。
  • 用現有的函式加上部份的參數列產生新的函式。

JavaScript 設計模式(筆記)_CH3_實字與建構式

Posted on 2017-08-17 | In 讀書筆記 |

物件實字

  • 實字符號模式(literal notation patterns)，可以有更簡潔、較不易出錯的物件定義方式。
  • JavaScript 的物件是一個 key-value pairs 的雜湊表，物件的值可以是原始型別或物件（稱為屬性）以及函式（稱為方法）
  • Javascript 建立的物件，在任何時間都是可變的（mutable）。

物件實字語法

用建構式創造物件

  • 可以使用類似於 Java 語言的 new 來建構物件
  • 實際上 JavaScript 是沒有 class 的，你不需要一個 class 的藍圖，這反而有很高的彈性。
  • 也可以使用內建的建構式，Object()、String()、Date()

建構式的陷阱

  • 可透過 constructor 屬性取得物件的建構式。
  • 建構原始物件，應該避免使用 new Object()，建議直接使用物件實字的方式。

自訂建構式函式

  • 建構式本身就是一個函式。
  • 注意使用 new 關鍵字，這會影響到建構式裡的 this 所參考的對象。
  • 使用原型 prototype 定義共用的函式或屬性，減少每次初始化物件所消耗的記憶體空間。
  • Object.create() 方法

建構式的回傳值

  • 一定會回傳一個物件。
  • 預設回傳 this 參考的物件。

強制 new 的模式

  • 忘記加入 new 關鍵字，this 會指向全域物件。
  • 應該盡量讓全域物件保持乾淨。

命名慣例

  • 依照函式命名規則來判斷是否為函式或建構式。

使用 that

  • 可以保證回傳一個物件。
  • 會失去與原型間的連結。

自我呼叫的建構式

  • 使用 instanceof 判斷，並自我呼叫。
  • 可以解決與原型間的連結。

陣列實字

  • 避免使用 new Array()。
  • 陣列建構式的陷阱。

檢查陣列

  • ECMAScript 5 的新方法 Array.isArray()。
  • 利用 Object.prototype.toString() 來檢查回傳字串"[object Array]"。

JSON

  • 一種資料傳輸格式，陣列跟物件的組合。
  • 使用 JSON.parse() 方法來轉換成 JSON 物件。
  • 使用 JSON.stringify() 序列化成 JSON 字串。

正規表示式實字

  • 使用 new RegExp() 建構式，需要考慮跳脫字元，會變得難以閱讀。
  • 使用正規表示實字，較短較好用。

正規表示式實字語法

  • g - 全域檢查
  • m - 檢查多行

  • i - 不區分大小寫

  • String.prototype.replace() 也可用正規表示式

  • 需要在執行期間才能得知正規表示式時，就可以使用 new RegExp() 建構式。

原始型別包裹

  • JavaScript 原始型別：數值、字串、布林、null 和 undefined。
  • 建構式 Number()、String()、Boolean()。
  • 包裹物件提供一些有用的方法，原始數值一樣可以使用，會變暫時轉型成物件來呼叫方法。
  • 用包裹物件建構出的物件可以擴充，原始數值不行。

錯誤物件

  • 內建的錯誤物件建構式 Error()、SyntaxError()、TypeError()。
  • 使用 throw 拋出錯誤物件，catch 接收
  • 也可以拋出自定義的物件，不一定要是個錯誤物件。
